# 编辑器

编辑器是 SuperLeaf 的主工作区，面向 LaTeX/Markdown 写作、批注审阅、PDF 预览和多人协作。

## 布局

| 区域 | 用途 |
|---|---|
| 左侧文件树 | 管理项目文件、文档和图片 |
| 中间编辑器 | CodeMirror 6，支持 LaTeX/Markdown |
| 批注栏 | 显示 Agent 建议、风险提示和用户评论 |
| 右侧面板 | 讨论区、团队管理、自动化、运行历史、版本 |
| 预览列 | LaTeX PDF 或 Markdown 预览 |

## 创建和编辑文档

1. 在文件树中新建 `.tex` 或 `.md`。
2. 点击文件进入编辑器。
3. 修改内容后，文档会自动保存。
4. 如果开启协作模式，文本通过 Yjs 同步，并由后端定期快照到数据库。

## LaTeX 预览

LaTeX 文档会通过后端调用 `latexmk` 编译：

```text
编辑器内容 → Backend /api/compile → latexmk → PDF → 前端 PDF.js 预览
```

如果预览失败：

- 确认本机安装了 `latexmk`。
- 查看错误日志中的缺包或语法错误。
- 先用最小文档验证环境。

## 拼写检查

编辑器内置轻量拼写检查，设计参考 Overleaf 的可见区域检查模型：

- 只检查当前可见编辑区域，避免大文档每次编辑都全量扫描。
- LaTeX 文档会跳过命令、引用键、标签、URL、数学公式和注释。
- Markdown 文档会跳过 fenced code、inline code、图片元数据和 URL。
- 拼写错误以波浪线标记，打开诊断提示后可以直接替换为建议词。
- 用户可以把项目术语或专有名词加入个人词典；词典按用户隔离。

当前后端使用本地词典引擎处理英文拼写和建议，后续可以在不改变前端交互的情况下替换为 Hunspell 或 LanguageTool。

## 选中文字运行 Agent

选中一段文本后，可以把它作为上下文交给 Agent 或 workflow。

常见用途：

- 检查逻辑跳跃
- 润色英文表达
- 压缩摘要
- 生成反驳意见
- 提取训练数据标记

Agent 输出会进入批注卡片，而不是直接覆盖正文。

## 批注卡片

批注卡片支持：

| 操作 | 说明 |
|---|---|
| 接受 | 将建议应用到文档 |
| 删除 | 移除该条卡片 |
| 追问 | 基于当前卡片继续和 Agent 对话 |
| 评价 | 记录质量和采用情况，用于后续训练数据 |

{: .note }
批注、对话和 workflow 运行都按 `user_id` 隔离。协作者共享文档内容，但不会自动看到彼此的 Agent 资产。

## 快捷键

常用快捷键细节目前先保留英文现有稿，见 [Editor Shortcuts](../../en/editor-shortcuts.md)。

编辑器内部基于 CodeMirror keymap，支持常见文本编辑行为。部分快捷键会随浏览器和操作系统略有差异。

## 多人协作

实时协作基于 Yjs：

- 远程光标和选择区通过 Awareness 协议显示。
- 文档内容冲突由 CRDT 自动合并。
- 后端定期把协作文档快照写回 SQLite。

详见 [多人协作](../collaboration/)。
