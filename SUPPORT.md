# Support

SuperLeaf is an open source project. Support is provided on a best-effort basis by maintainers and contributors.

## Where to Get Help

- Setup and usage questions: open a GitHub issue.
- Reproducible bugs: use the bug report template.
- Feature ideas: use the feature request template.
- Security issues: follow `SECURITY.md` and avoid public disclosure.

Before opening an issue, check:

- `README.md`
- `docs/getting-started/install.md`
- `docs/getting-started/first-run.md`
- `docs/troubleshooting/README.md`
- Existing GitHub issues

## Useful Details to Include

- Operating system and version.
- Node.js, Python, and `uv` versions.
- The command you ran.
- Relevant logs from `logs/latest/`.
- Browser and console errors for frontend issues.
- Provider configuration type: Nanobot, Dify, Claude API, or offline.
- Whether the issue affects one document, one project, or every project.

## Project Boundaries

Maintainers can help with SuperLeaf setup, configuration, bugs, and documented workflows. They generally cannot provide private deployment operations, academic writing review, third-party provider account support, or debugging for heavily modified forks unless a maintainer chooses to do so.
